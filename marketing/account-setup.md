## Secure Setup Rules

- Use an email alias controlled by the project owner, not a disposable inbox.
- Use unique generated passwords in a password manager.
- Enable two-factor authentication during setup when available.
- Store backup codes in the password manager, not in this repository.
- Do not connect personal contacts during onboarding.
- Do not store OAuth tokens, API keys, recovery phrases, phone numbers, or
  passwords in project files.
- If a platform asks for phone, CAPTCHA, email verification, or identity
  confirmation, the owner must complete that step directly in the browser.
